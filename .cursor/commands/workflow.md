Execute the complete interactive git workflow following the process defined in @.cursor/rules/workflow.mdc

**Follow these steps in order:**

1. **Analyze Recent Changes** - Check git status, diffs, and recent commits
2. **Branch Management** - Ensure on appropriate branch (ask if unclear)
3. **Test Changes** - Run checks, type checking, and tests
4. **Documentation Check** - Find and update related docs if needed
5. **Commit in Meaningful Chunks** - Group related changes, ask if unclear
6. **Handle Errors/Issues** - Ask user if anything goes wrong
7. **Push Changes** - Ask for confirmation before pushing
8. **Create Pull Request** - Create PR with proper description
9. **Summary Report** - Provide complete summary at end

**Critical Rules:**
- Ask developer if anything is unclear
- Otherwise proceed autonomously
- Always test before committing
- Always use proper commit message format
- Always provide summary at end

Reference: @.cursor/rules/workflow.mdc for complete details and examples.
